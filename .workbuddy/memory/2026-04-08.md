## Bug 修复：MySQL DDL 不显示

修复了 `MySQLMetadataProvider.getTableDDL` 方法中的严重 Bug：
- **根因 1**：用 `StringJoiner` 构建的字符串是 `` `catalog.tableName` ``，缺少 `SHOW CREATE TABLE` 关键字，生成的不是合法 SQL
- **根因 2**：使用了 `PreparedStatement`，而 `SHOW CREATE TABLE` 不支持参数化查询，应使用 `Statement`
- **修复**：重写方法，正确拼接 `SHOW CREATE TABLE \`catalog\`.\`tableName\`` 并使用 `Statement.executeQuery()` 执行
- 同时移除了遗留的 `System.out.println` 调试代码和无用的 `StringJoiner` import

## Bug 修复：连接测试按钮点击第二次界面卡死

- **根因 1**：`testConnection()` 在 UI 线程同步执行，Druid 连接超时默认 60 秒，造成 UI 冻结
- **根因 2**：`DataSourceManager.createDataSource` 连接失败时仍将 DataSource 放入缓存，第二次点击直接取到失败的连接池，再次触发 60 秒等待
- **修复 1**：`DataSourceManager` 新增 `testConnectionDirect()` 方法，不走缓存，临时创建连接池（`maxWait=5s`，`initialSize=0`），用完立即关闭
- **修复 2**：`MySQLConfigurationController.testConnection()` 改为后台线程异步执行，测试期间按钮禁用并显示"测试中..."，完成后在 UI 线程弹窗提示结果
